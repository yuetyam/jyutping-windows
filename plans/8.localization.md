# Localization Roadmap

This plan covers localizing the Windows Jyutping IME user-facing strings for:

- `en` or `en-US`
- `zh-HK`
- `zh-TW`
- `zh-CN`

The goal is to make the IME present native, predictable strings in Windows UI,
TSF profile registration, language bar items, preserved-key descriptions,
and resource metadata while keeping input behavior unchanged.

Out of scope for this roadmap:

- README and public documentation localization.
- Installer localization.
- Website, release notes, changelog, GitHub issue templates, and other
  non-IME localization work.

## Current Baseline

Most visible strings are currently English or Traditional Chinese literals in
code and resources.

Relevant files:

- `Jyutping\Jyutping.rc`
  - Version resource strings.
  - Icon and string table resources.
  - Current `IDS_INPUT_MODE` string.
- `Jyutping\resource.h`
  - Resource identifiers for string table values.
- `Jyutping\Register.cpp`
  - `TEXTSERVICE_DESC`, currently a hard-coded profile name.
- `Jyutping\Globals.cpp`
  - Preserved-key descriptions.
  - Language bar item descriptions.
  - Window class names that should not be localized.

Current examples:

- `TEXTSERVICE_DESC = L"Jyutping"`
- `Input Mode (Cantonese/ABC, Shift)`
- `Character Form (Half-width/Full-width, Shift+Space)`
- `Punctuation Form (Cantonese/English, Ctrl+.)`
- `Traditional Characters (Ctrl+Shift+1)`
- `Simplified Characters (Ctrl+Shift+4)`
- Version metadata:
  - `FileDescription = 粵語拼音輸入法`
  - `ProductName = 粵拼`

## Design Goals

- Keep localization separate from input-engine logic.
- Use Windows resource loading for user-facing UI strings.
- Avoid localizing protocol identifiers, registry paths, database paths,
  SQLite SQL, class names, GUID-related names, and log operation labels unless
  there is a clear user-facing reason.
- Make English the fallback language.
- Keep Traditional Chinese for `zh-HK` and `zh-TW`, but allow wording
  differences between Hong Kong and Taiwan.
- Use Simplified Chinese for `zh-CN`.
- Keep product identity consistent across languages.
- Make missing translations obvious during development.
- Keep this roadmap scoped to the IME DLL and its resources.

## Language Policy

Use these locale roles:

- `en-US`: English fallback and development source.
- `zh-HK`: Primary Traditional Chinese localization for Cantonese users.
- `zh-TW`: Traditional Chinese localization with Taiwan wording where needed.
- `zh-CN`: Simplified Chinese localization.

Use `en-US` internally for resource language blocks because Windows resources
identify a concrete language and sublanguage.

## Phase 1: String Inventory

Classify all string literals before changing code.

Localize:

- TSF profile display name.
- Preserved-key descriptions.
- Language bar item descriptions.
- String table values.
- File description and product name in version resources, if supported through
  language-specific version blocks.

Do not localize:

- Window class names:
  - `Jyutping.CandidateWindow`
  - `Jyutping.ShadowWindow`
  - `Jyutping.ScrollBarWindow`
- Registry paths:
  - `Software\Jyutping\Settings`
- File and directory names:
  - `%LOCALAPPDATA%\Jyutping`
  - `memory.sqlite3`
  - `ime.sqlite3`
- SQLite table and column names.
- Debug log operation labels, unless a later diagnostic UX requires localized
  logs.
- Input keys, romanization strings, database content, and candidate comments.

Suggested audit commands:

```powershell
rg -n 'L"|TEXT\(|STRINGTABLE|VALUE "|SysAllocString' Jyutping
rg -n 'Description|ProductName|FileDescription|RegisterProfile|InitPreservedKey' Jyutping
```

Completion criteria:

- Every user-facing string has an owner and resource ID.
- Every non-localized string has a reason to remain literal.

## Phase 2: Resource Model

Move user-facing strings into Windows resources.

Use separate included resource files for localized strings.

Recommended files:

- `Jyutping\Resources\Strings.en-US.rc2`
- `Jyutping\Resources\Strings.zh-HK.rc2`
- `Jyutping\Resources\Strings.zh-TW.rc2`
- `Jyutping\Resources\Strings.zh-CN.rc2`

Resource model:

- Keep `Jyutping\Jyutping.rc` as the top-level resource script.
- Include the four `.rc2` files from `Jyutping.rc`.
- Put language-specific `STRINGTABLE` blocks in the `.rc2` files.
- Use numeric `IDS_*` identifiers in `resource.h`.
- Add a small helper for loading strings from the module resource table.
- Keep icons and non-localized structural resources in `Jyutping.rc`.

Recommended resource IDs:

```cpp
#define IDS_TEXTSERVICE_DESC                         20
#define IDS_INPUT_MODE_DESCRIPTION                   21
#define IDS_CHARACTER_FORM_DESCRIPTION               22
#define IDS_PUNCTUATION_FORM_DESCRIPTION             23
#define IDS_CHARACTER_VARIANT_TRADITIONAL            24
#define IDS_CHARACTER_VARIANT_HONG_KONG              25
#define IDS_CHARACTER_VARIANT_TAIWAN                 26
#define IDS_CHARACTER_VARIANT_SIMPLIFIED             27
#define IDS_LANGBAR_INPUT_METHOD_MODE                28
#define IDS_FILE_DESCRIPTION                         29
#define IDS_PRODUCT_NAME                             30
```

Keep existing IDs where practical, but rename ambiguous identifiers if the
resource table becomes easier to maintain.

Completion criteria:

- All planned localizable strings have stable resource IDs.
- `Jyutping.rc` includes the locale-specific `.rc2` files.
- English resources compile and match current behavior.

## Phase 3: Resource Loading Helper

Add a small localization helper.

Suggested files:

- `Jyutping\Localization.h`
- `Jyutping\Localization.cpp`

Suggested API:

```cpp
namespace Localization
{
    std::wstring LoadString(UINT resourceId);
    std::wstring LoadStringOrFallback(UINT resourceId, PCWSTR fallback);
}
```

Implementation notes:

- Load from `Global::dllInstanceHandle`.
- Use `LoadStringW`.
- Return the fallback when loading fails.
- Keep the helper independent from TSF and candidate UI classes.
- Do not cache at first unless profiling or repeated call sites justify it.

Update project files:

- `Jyutping\Jyutping.vcxproj`
- `Jyutping\Jyutping.vcxproj.filters`

Completion criteria:

- A single helper is used by code that needs localized strings.
- Missing resources fall back safely to English literals.

## Phase 4: Localize TSF Registration

Update registration to use localized resources.

Target file:

- `Jyutping\Register.cpp`

Current behavior:

- `RegisterProfile()` passes `TEXTSERVICE_DESC` to TSF.
- `RegisterServer()` writes `TEXTSERVICE_DESC` under the COM CLSID key.

Planned behavior:

- Load `IDS_TEXTSERVICE_DESC` during registration.
- Pass the localized text service name to `RegisterProfile()`.
- Write the same localized value as the default CLSID description.
- Keep the CLSID, profile GUID, language ID, icon path, and icon index
  unchanged.

Important behavior to verify:

- Registration happens when `DllRegisterServer` runs, so the profile name may
  reflect the registering user's UI language at that moment.
- If Windows does not automatically refresh an already registered profile name
  after language changes, document that reinstall or repair may be required to
  refresh registration-time strings.

Completion criteria:

- The Windows input profile display name is localized after fresh install.
- Registration succeeds when the resource load fails by falling back to English.

## Phase 5: Localize Preserved-Key And Language Bar Strings

Update hard-coded descriptions in `Globals.cpp`.

Current strings include:

- `InputMethodModeDescription`
- `CharacterFormDescription`
- `PunctuationFormDescription`
- `TraditionalCharacterVariantDescription`
- `HongKongCharacterVariantDescription`
- `TaiwanCharacterVariantDescription`
- `SimplifiedCharacterVariantDescription`
- `LangbarInputMethodModeDescription`

Recommended approach:

- Replace global constant string arrays with resource IDs or loaded string
  storage.
- Load descriptions during global initialization or immediately before
  preserved-key registration.
- Keep shortcut notation consistent across locales.

Suggested translation style:

- English: `Input Mode (Cantonese/ABC, Shift)`
- `zh-HK`: `輸入模式（粵語/英文，Shift）`
- `zh-TW`: `輸入模式（粵語/英文，Shift）`
- `zh-CN`: `输入模式（粤语/英文，Shift）`

Correct current shortcut labels while localizing:

- Hong Kong variant should say `Ctrl+Shift+2`.
- Taiwan variant should say `Ctrl+Shift+3`.
- Simplified variant should say `Ctrl+Shift+4`.

Completion criteria:

- Preserved-key registration uses localized descriptions.
- Language bar text uses localized descriptions.
- Shortcut labels are correct in all locales.

Target file:

- `ProductName`

Fields to keep stable:

Suggested product naming:

- `zh-HK`:
  - `ProductName`: `粵拼`
  - `FileDescription`: `粵拼粵語輸入法`
- `zh-TW`:
  - `ProductName`: `粵拼`
  - `FileDescription`: `粵拼粵語輸入法`
- `zh-CN`:
  - `ProductName`: `粤拼`
  - `FileDescription`: `粤拼粤语输入法`

Completion criteria:

  blocks.
- English metadata remains available as fallback.

## Phase 7: Translation Table

Create a checked-in translation source table.

Suggested file:

- `plans\localization-strings.md`

Suggested columns:

- Resource ID.
- English.
- `zh-HK`.
- `zh-TW`.
- `zh-CN`.
- Notes.

Rules:

- Update the table before editing `.rc` resources.
- Mark strings that intentionally match across `zh-HK` and `zh-TW`.
- Note shortcut labels separately from translatable words.

Completion criteria:

- Reviewers can inspect translations without reading `.rc` syntax.
- Missing translations are visible in one file.

## Phase 8: Verification

Build checks:

```powershell
msbuild Jyutping.sln /p:Configuration=Debug /p:Platform=x64
msbuild Jyutping.sln /p:Configuration=Release /p:Platform=x64
msbuild Jyutping.sln /p:Configuration=Debug /p:Platform=ARM64
msbuild Jyutping.sln /p:Configuration=Release /p:Platform=ARM64
```

Manual checks:

1. Set Windows display language to English.
2. Install/register the IME.
3. Confirm profile name, language bar text, preserved-key descriptions, file
   properties, and other IME-owned UI strings are English.
4. Repeat for `zh-HK`.
5. Repeat for `zh-TW`.
6. Repeat for `zh-CN`.
7. Confirm candidate text, romanization comments, raw input, database lookup,
   and input memory behavior are unchanged by UI language.
8. Confirm unsupported or missing language resources fall back to English.
9. Confirm all strings fit in Windows UI surfaces.

Regression checks:

- TSF registration and unregistration still work.
- Preserved keys still register and trigger correctly.
- Language bar icons and toggles still work.
- Candidate window rendering is unchanged except for localized labels where
  applicable.

## Implementation Order

1. Inventory user-facing strings.
2. Add resource IDs.
3. Add separate included `.rc2` files for `en-US`, `zh-HK`, `zh-TW`, and
   `zh-CN`.
4. Add the localization loading helper.
5. Move TSF registration strings to resources.
6. Move preserved-key and language bar strings to resources.
7. Add `zh-HK`, `zh-TW`, and `zh-CN` resource blocks.
8. Localize version metadata.
9. Run the full build and manual locale verification matrix.

## Completion Criteria

- All supported locales have resource strings for user-facing IME UI.
- English is the fallback language.
- TSF profile name is localized on fresh registration.
- Preserved-key and language bar descriptions are localized.
- Version metadata is localized where Windows supports it.
- Input behavior and candidate generation are unchanged.
