# Installer And Uninstaller Roadmap

This plan covers the two Inno Setup installer packages:

- `jyutping-v0.1.0-x64.exe`
- `jyutping-v0.1.0-arm64.exe`

The installers must copy the correct DLLs and `ime.sqlite3`, register the TSF
COM server/profile, refuse the wrong CPU platform, and provide a stable
Windows uninstall entry backed by Inno Setup's generated uninstaller.

## Build Inputs

Use these release outputs as installer payloads:

- x64 DLL: `x64\Release\Jyutping.dll`
- x86 DLL: `Win32\Release\Jyutping.dll`
- ARM64 root DLL: `ARM64EC\Release\Jyutping.dll`
- database: `Jyutping\Resources\ime.sqlite3`

The ARM64 root DLL is the ARM64X build. It is the DLL that should be registered
as `Program Files\Jyutping\Jyutping.dll` on ARM64 Windows. Do not install a
separate x64 DLL payload on ARM64 Windows; x64 clients use the registered
ARM64X root DLL. On ARM64 Windows, supports 64-bit clients only: native ARM64
clients and emulated x64 clients. Do not install or register an x86 legacy
fallback on ARM64 Windows.

CI already builds `Win32`, `x64`, `ARM64`, and `ARM64EC` for both `Debug` and
`Release`.

## Target Layouts

### x64 Installer Payload

```text
jyutping-v0.1.0-x64.exe
|
+-- x64/
|   +-- Jyutping.dll
+-- x86/
|   +-- Jyutping.dll
+-- ime.sqlite3
```

Install result:

```text
Program Files/Jyutping/
|   Jyutping.dll
|   ime.sqlite3
|   unins000.exe

Program Files (x86)/Jyutping/
|   Jyutping.dll
|   ime.sqlite3
```

### ARM64 Installer Payload

```text
jyutping-v0.1.0-arm64.exe
|
+-- Jyutping.dll
+-- ime.sqlite3
```

Install result:

```text
Program Files/Jyutping/
|   Jyutping.dll
|   ime.sqlite3
|   unins000.exe
```

## Roadmap

## 1. Add Packaging Files

Create an `installer/` directory:

```text
installer/
|   Build-Installers.ps1
|   Jyutping.common.iss
|   Jyutping-x64.iss
|   Jyutping-ARM64.iss
|   stage/
```

`Build-Installers.ps1` should:

- build `Release|x64`, `Release|Win32`, and `Release|ARM64EC`;
- stage files into the payload layouts above;
- copy `ime.sqlite3` once into each staged installer root;
- fail if any required DLL or database payload is missing;
- call Inno Setup to create both installer EXEs.

## 2. Use Inno Setup Uninstaller

Use Inno Setup's generated `unins000.exe` instead of a dedicated native
uninstaller launcher. `UninstallFilesDir={app}` keeps the Inno uninstaller in
the primary install directory. The stable uninstall entry point is the Windows
Apps & Features entry and Inno uninstall registry command.

## 3. Write The Inno Setup Scripts

Shared settings:

- `AppName=Jyutping`
- stable `AppId` distinct from the TSF CLSID
- `AppVersion={#AppVersion}`
- `DefaultDirName={autopf64}\Jyutping`
- `PrivilegesRequired=admin`
- output filenames:
  - `jyutping-v{#AppVersion}-x64.exe`
  - `jyutping-v{#AppVersion}-arm64.exe`

x64 installer:

- allow x64 Windows only;
- install x64 files to `{autopf64}\Jyutping`;
- install x86 files to `{autopf32}\Jyutping`;
- register the x64 DLL in the 64-bit COM view;
- register the x86 DLL in the 32-bit COM view.

ARM64 installer:

- allow ARM64 Windows only;
- install `Jyutping.dll` to `{autopf64}\Jyutping`;
- register the ARM64X root DLL as the 64-bit COM server;
- do not install or register an x86 DLL.

## 4. Make Registration Safe

Before packaging, make sure registration and unregistration are repeatable:

- `DllRegisterServer` succeeds when the profile/category already exists;
- `DllUnregisterServer` succeeds when entries are already absent;
- failed registration rolls back cleanly;
- logs include enough detail for install failures.

## 5. Installation Behavior

Both installers should:

- refuse unsupported CPU architectures before copying files;
- require elevation;
- unregister an existing installation before replacing DLLs;
- copy all payload files;
- register their supported DLL payloads;
- leave `%LOCALAPPDATA%\Jyutping` user data untouched.

If a DLL is in use, request a restart instead of force-closing client apps.

## 6. Uninstallation Behavior

The uninstaller should:

- unregister the primary 64-bit DLL;
- unregister the x86 DLL only when installed by the x64 package;
- remove installed files from each package's install directories;
- remove empty install directories;
- leave logs and input memory under `%LOCALAPPDATA%\Jyutping`.

## 7. Release Automation

For release builds:

- sign each staged `Jyutping.dll`;
- build both Inno installers;
- sign the final installer EXEs;
- generate SHA-256 checksums;
- upload installers and checksums as release artifacts.

## 8. Test Checklist

x64 Windows:

- x64 installer succeeds.
- ARM64 installer refuses.
- x64 and x86 DLLs are installed with `ime.sqlite3` beside them.
- 64-bit and 32-bit clients can activate Jyutping.
- the Windows uninstall entry removes the installation.

ARM64 Windows:

- ARM64 installer succeeds.
- x64 installer refuses.
- ARM64X root DLL is installed with `ime.sqlite3` beside it.
- ARM64 and x64 clients can activate Jyutping.
- x86 clients are unsupported and have no ARM64 installer fallback.
- the Windows uninstall entry removes the installation.

Upgrade/repair:

- reinstall repairs missing files;
- upgrade replaces DLLs and preserves user data;
- uninstall after upgrade removes the current files and registry entries.

## Acceptance Criteria

- Both installer filenames match the requested versioned names.
- Both installers refuse the wrong CPU platform.
- Installed paths match the requested layouts.
- Every installed DLL has `ime.sqlite3` beside it.
- The TSF profile appears after install and disappears after uninstall.
- x64 and x86 client activation works on x64 Windows.
- ARM64 and x64 client activation works on ARM64 Windows.
- ARM64 Windows does not install or register an x86 fallback.
- The Inno Setup uninstaller works silently and interactively.
- Reinstall, repair, upgrade, and uninstall are repeatable without manual
  registry cleanup.
